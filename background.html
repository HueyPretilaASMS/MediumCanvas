Background

The Basis of MediumCanvas

The Problem

When doing a collaboration in the creation of art with other people digitally, there are many various art programs to facilitate this collaboration. However, rarely are they able to efficiently exert this collaboration because of their complicated method in connecting multiple people through files. MediumCanvas instead will run on servers as the means to connect people, allowing artists to have quicker access to the service.

The Solution

Our innovation is a program that allows more than one artist to work at an art project at any given time. This is an ever-present issue where artists cannot collaborate with each other effectively since similar programs are complicated to use and are not accessible to current artists. We decided to take a different method in creating this program by creating a service that would allow people to collaborate through any internet connection. Many opportunities can stem from this program: from enabling freelance artists to collaborate to feedback effectively given by viewing the entire process in real t10me, to businesses in efficiently creating materials for their services. We hope in the future that we can add additional features such as pin to taskbar and different default layout options.

Development of MediumCanvas

Design of the Program

For the design, we collated ideas from already existing interfaces Krita and Paint Tool Sai. We wanted to emulate the best parts of these programs. We wanted our layout to be minimal and efficient-diminishing confusion from the user of the placement of tools. We kept the top task bars similar to Krita, having the same order. On the right side there is a section that says what people are there similar to that of Google Doc. We decided for the options bar below the top one would only include opacity, size, switching colors option, and drop down menu for the tools for those who do not like the tool bar. We decided for small number of options to keep it minimal and not cluttered. We made the the options for brushes and pens on the right tool bar with the color. This will make it bigger and prevent having to go to the second top bar, saving time when adjusting color and changing tool. We also made the layers in a separate tab from the color wheel to leave room for the brushes. This decision would prevent the layers to be squished and allow elements within the layer to be seen easier beacuse of its bigger size. So that the color weel and layers could be seen at the same time, we will also add the option for the layer tab to be placed in either left or right side. The color wheel and layers tab being side by side will not be a default layout as it is in Paint Tool Sai because it will hinder the size of the canvas. Prototyping of the Program

Prior to the latest prototype, many iterations of the same program was being sought out. In each one, the focus was on performance and usability. Several sketches were created before coding to understand the appropriate layouts for the program and the website. These sketches combined the strengths and weaknesses of many well established programs and websites such as Photoshop’s User Interface and Mozilla Firefox’ homepage. In this way, insight was gained to the elements that were successful and lacking before coding- preventing long stretches of trial and error when it came to the design. However, there was still plenty of experimentation in maximizing the performance and usability of the programing; since there were no references for coding, forcing the programmer to do the code from scratch. One notable example is a mathematical equation the programer needed to solve to make the lines continuous and not dotted.
